개요

JavaScript 자동 스크롤이란?

window.scrollBy()와 requestAnimationFrame()을 결합하면 부드럽고 제어 가능한 자동 스크롤을 구현할 수 있습니다.

매 프레임마다 일정 픽셀씩 스크롤하고, 페이지 맨 아래에 도달하면 자동으로 멈춥니다.

let scrollId = null;
let speed = 5;

function autoScroll() {
  const maxScroll =
    document.body.scrollHeight - window.innerHeight;
  if (window.scrollY >= maxScroll) {
    stopScroll(); return;
  }
  window.scrollBy(0, speed);
  scrollId = window.requestAnimationFrame(autoScroll);
}

핵심 API

사용된 주요 Web API

window.scrollBy(x, y) — 현재 위치 기준 상대적으로 스크롤

requestAnimationFrame(cb) — 60fps 렌더링 주기에 맞춰 콜백 반복 호출

cancelAnimationFrame(id) — 예약된 프레임 콜백 취소

document.body.scrollHeight — 페이지 전체 높이

window.scrollY — 현재 스크롤 위치

성능

requestAnimationFrame vs setInterval

setInterval은 백그라운드 탭에서도 계속 실행되어 CPU를 낭비합니다.

requestAnimationFrame은 탭이 숨겨지면 자동 일시정지되고 VSync에 맞춰 더 부드러운 애니메이션을 보장합니다.

이벤트

사용자 휠 감지 → 정지

window.addEventListener('wheel', () => {
  if (scrollId) stopScroll();
});

진행률

스크롤 진행률 계산

window.addEventListener('scroll', () => {
  const pct =
    (window.scrollY /
    (document.body.scrollHeight - window.innerHeight))
    * 100;
  progressBar.style.width = pct + '%';
});

속도 제어

슬라이더로 실시간 조절

슬라이더 값을 speed 변수에 실시간 반영하여 scrollBy의 이동량을 동적으로 변경합니다.

<input type="range" min="1" max="20"
  oninput="speed = +this.value" />

💬 과제 소감

Task 1을 마치며

이번 Task에서 requestAnimationFrame을 직접 활용하면서, setInterval과의 차이를 코드로 체감할 수 있었습니다. 브라우저의 렌더링 주기(VSync)에 맞춰 동작하기 때문에 훨씬 부드럽고, 탭이 숨겨지면 자동으로 일시정지된다는 점이 인상적이었습니다.

속도 슬라이더와 진행률 바를 함께 구현하면서 사용자 인터랙션의 중요성도 새삼 느꼈습니다. 단순히 스크롤을 자동화하는 것에서 그치지 않고, 사용자가 제어권을 가질 수 있도록 설계하는 것이 좋은 UX임을 배웠습니다.
